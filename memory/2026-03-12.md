### Quality Issue - Root Cause Analysis
- When VAPI uses Vonage as telephony provider, calls route through Vonage's PSTN gateway (extra hop)
- VAPI native numbers use their own Twilio-based telephony (direct, optimized)
- The additional SIP hop through Vonage adds latency and potential codec transcoding
- No `TransportConfigurationVonage` schema exists in VAPI's API (only `TransportConfigurationTwilio`)
- This means there are NO configurable audio/codec settings for Vonage calls in VAPI
- The quality difference is architectural, not a configuration problem
