## How We Work

### AI-First Approach

**What This Means:**
- AI agents draft code, docs, and project plans first
- Humans (you) review, refine, and improve AI output
- We move FAST because we start at 70% complete, not 0%

**Your Role:**
- **Review:** Check AI-generated code for bugs, edge cases, security issues
- **Refine:** Improve structure, add error handling, optimize performance
- **Build:** Handle complex features AI cannot do (yet)
- **Quality Gate:** You are the final check before shipping to clients

**Why We Do This:**
- Speed: Deliver in days, not weeks
- Cost: Keep prices competitive
- Scale: Handle more clients without hiring armies of people

**Important:** We are not replacing humans with AI. We are augmenting humans with AI. You are the expert; AI is your assistant.

### Core Values

1. **Speed Over Perfection:** Ship working solutions fast, iterate based on feedback
2. **Clarity Over Cleverness:** Simple code > clever code
3. **Client Results > Internal Process:** If it works for the client, it works
4. **Async Communication:** Write clear updates, avoid unnecessary meetings
5. **Ownership:** If you see a problem, you own fixing it (or flagging it)

## Quality Standards

### Code Quality (Python Devs)

**Required:**
- Code runs without errors in clean environment
- Clear variable names (no `x`, `temp`, `data1`)
- Comments for complex logic
- Error handling (try/except where appropriate)
- No hardcoded credentials (use .env files)

**Nice to Have:**
- Type hints for functions
- Unit tests (if time permits)
- README for setup instructions

**Not Required:**
- Perfect architecture (we iterate, not over-engineer)
- 100% test coverage (focus on critical paths)

### Documentation Quality (Writers)

**Required:**
- Written for the target audience (technical vs non-technical)
- Step-by-step instructions with screenshots
- Clear headers, scannable structure
- No jargon without explanation
- Tested by someone else (it should work for a beginner)

**Nice to Have:**
- Video walkthroughs
- Troubleshooting section
- Visual diagrams

### VA/PM Quality

**Required:**
- Client emails are professional, friendly, clear
- Projects updated daily in tracker
- Meetings have agendas + notes
- Follow-ups happen within 24 hours
- Files organized (no "Untitled" docs)

**Nice to Have:**
- Proactive problem-solving
- Client anticipation (spot issues before they complain)

## Communication Protocols

### Response Time Expectations

**Urgent (client-facing issues):** <2 hours during your work hours
**Normal requests:** <4 hours same business day
**Non-urgent questions:** <24 hours

**If You Will Be Unavailable:** Post in Slack with timeframe.

### Daily Updates (Required)

Post in Slack #updates channel:

**Format:**
```
[Your Name] - [Date]
✅ Completed: [what you finished]
🚧 In Progress: [what you are working on]
🚨 Blockers: [anything stopping you]
⏰ Hours: [hours worked today]
```

**Example:**
```
Avery - Feb 20
✅ Completed: Onboarding guide draft, hiring plan finalized
🚧 In Progress: Job postings + freelancer research
🚨 Blockers: None
⏰ Hours: 6
```

### Weekly Sync (15-30 min)

**When:** Every Monday (or your first work day of week)
**Format:** Quick Zoom call or async Loom video
**Topics:**
- Last week wins
- This week priorities
- Any blockers or questions

### Asking Questions

**Good Question:**
"I am building the API integration for Client X. I am getting a 401 error on the POST request. I have checked the API key and endpoint. Here is the code [link]. Any ideas?"

**Bad Question:**
"The API does not work. Help."

**Rule:** Explain what you tried before asking. We are happy to help, but do your homework first.

## Working With AI Agents

### How AI Agents Work at ClawOps

**AI Generates First Draft:**
- Code skeletons, boilerplate, basic logic
- Documentation outlines, first drafts
- Research summaries, data collection

**You Refine and Validate:**
- Check for errors, edge cases
- Add complexity AI missed
- Ensure it meets client needs
- Test thoroughly

### Common AI Mistakes to Watch For

**Code:**
- Missing error handling
- Hardcoded values that should be variables
- Inefficient loops (AI loves nested loops)
- Outdated library syntax (AI training data lags)

**Documentation:**
- Too verbose or too brief
- Missing context (assumes reader knows more than they do)
- Generic examples that do not match real use case

**Your Job:** Spot these, fix them, teach AI over time (we log improvements).

### How to Give Feedback on AI Output

**Format:**
```
AI Draft: [link to code/doc]
Issues Found: [specific problems]
Fixed: [what you changed]
Time Saved: [estimate vs building from scratch]
```

**Why This Matters:** We track AI effectiveness to improve our process.

## Project Workflow

### Standard Project Flow

1. **Kickoff:** Blake or PM shares client brief
2. **Planning:** You review requirements, ask clarifying questions
3. **Development:** You build (with AI assistance), test locally
4. **Review:** You submit for internal QA check
5. **Delivery:** PM shares with client, you handle revisions if needed
6. **Wrap-Up:** Document what you built, update knowledge base

### Your Responsibilities by Phase

**Planning:**
- Read the brief thoroughly
- Ask questions BEFORE starting (saves rework)
- Estimate time realistically (under-promise, over-deliver)

**Development:**
- Commit code regularly (not one giant commit at the end)
- Test edge cases (what breaks this?)
- Write clear commit messages

**Review:**
- Self-QA first (run it yourself as if you are the client)
- Submit with testing notes ("I tested X, Y, Z scenarios")

**Delivery:**
- Quick turnaround on client feedback (<24 hours if possible)
- Be gracious about revisions (clients change their minds, it is normal)

**Wrap-Up:**
- Document anything non-obvious
- Share learnings with team

## FAQ

### What if I disagree with how something should be built?

**Answer:** Share your perspective! We value your expertise. Format: "Here is what was requested. Here is why I think approach X is better. What do you think?" Blake will decide, but your input shapes the decision.

### What if a client asks me to do something outside my contract?

**Answer:** Loop in Blake or PM immediately. Do NOT agree to scope changes directly with clients.

### What if I make a mistake that affects a client?

**Answer:** Tell us immediately. We will fix it together. Hiding mistakes makes them worse. Transparency = trust.

### Can I work on other clients while contracted with ClawOps?

**Answer:** Yes, as long as you meet your committed hours and deadlines. Just avoid conflicts of interest (do not work for competitors or poach our clients).

### What if I need time off?

**Answer:** Give 1 week notice if possible. Post in Slack, adjust your availability. We are flexible, just communicate clearly.

### How do I grow within ClawOps?

**Answer:** Deliver great work consistently. As we scale, we will need team leads, specialists, and full-time roles. Contractors who prove themselves get first dibs.

### What if I want to leave?

**Answer:** Give 1 week notice. We will wrap up your projects and part on good terms. No hard feelings (freelancing is about flexibility).
