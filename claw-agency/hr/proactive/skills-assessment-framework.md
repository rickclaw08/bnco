# ClawOps — AI Automation Contractor Skills Assessment Framework

## Purpose

Standardized evaluation of contractors before engagement. Score candidates across technical, soft, and domain skills to ensure quality hires.

---

## Assessment Process

1. **Resume / Portfolio Review** (15 min)
2. **Technical Screening Call** (30 min)
3. **Practical Assessment** (take-home, 2-4 hours)
4. **Culture & Communication Fit** (30 min call)
5. **Reference Check** (optional but recommended for long-term engagements)

---

## Scoring Rubric

Each skill: **1** (None) → **2** (Basic) → **3** (Competent) → **4** (Strong) → **5** (Expert)

**Minimum to engage:** Average ≥ 3.0 overall, no critical skill below 2.

## Practical Assessment Options

Choose one based on the role:

### Option A: Automation Build (General)
> Build a simple automation that takes a CSV of company names, enriches each with data from a public API, and outputs a formatted report. Time limit: 3 hours.

### Option B: AI Integration
> Create a basic RAG chatbot that answers questions from a provided set of documents. Evaluate retrieval accuracy and response quality. Time limit: 4 hours.

### Option C: Workflow Design (No-Code/Low-Code)
> Design an n8n/Make workflow that monitors an inbox, classifies emails by intent, and routes them to appropriate channels. Provide screenshots and documentation. Time limit: 2 hours.

**Evaluation criteria:** Functionality (40%), Code quality (25%), Documentation (20%), Error handling (15%)

### Decision Thresholds
- **≥ 4.0** → Strong hire, prioritize engagement
- **3.5 – 3.9** → Good fit, proceed with standard terms
- **3.0 – 3.4** → Conditional, consider for junior/support roles
- **< 3.0** → Pass

---

## Red Flags (Auto-Decline)
- Cannot explain their own past work
- No version control experience
- Poor communication during the process itself
- Misrepresents skills or experience
- Unwilling to sign NDA/contractor agreement
