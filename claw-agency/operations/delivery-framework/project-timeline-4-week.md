## Sprint Overview

| Week | Focus | Key Deliverable |
|------|-------|----------------|
| Week 1 | Discovery, setup, start building | Requirements locked, environment ready, first component started |
| Week 2 | Core build | Primary automations working |
| Week 3 | Integration, testing, iteration | End-to-end system working, client feedback round |
| Week 4 | QA, documentation, handoff | Fully tested, documented, delivered |

---

## Weekly Breakdown

### Week 1: Foundation

**Goals:** Lock requirements, set up everything, start building.

- [ ] Day 1: Kickoff call, collect access/credentials
- [ ] Day 1: Set up dev environment, project board, file structure
- [ ] Day 2: Deep-dive into client's existing systems
- [ ] Day 2-3: Build Component 1 (highest priority automation)
- [ ] Day 4: Test Component 1 with sample data
- [ ] Day 5: Begin Component 2
- [ ] Day 5: Send Weekly Status Update

**Client touchpoint:** Kickoff call (Day 1), async update (Day 3), status email (Day 5)

---

### Week 2: Core Build

**Goals:** Build the primary automations. Get the system working end-to-end, even if rough.

- [ ] Day 6-7: Complete Component 2
- [ ] Day 7-8: Build Component 3
- [ ] Day 8-9: Begin integrating components
- [ ] Day 9: Test integrated workflow with sample data
- [ ] Day 10: Demo rough version to client for early feedback
- [ ] Day 10: Send Weekly Status Update

**Client touchpoint:** Quick demo or Loom video (Day 10), status email (Day 10)

---

### Week 3: Integration and Iteration

**Goals:** Everything connected. Client feedback incorporated. Edge cases handled.

- [ ] Day 11: Incorporate client feedback from Week 2 demo
- [ ] Day 11-12: Build remaining components (if any)
- [ ] Day 12-13: Full integration testing
- [ ] Day 13-14: Error handling and edge cases
- [ ] Day 14: Run against real/production data (with client approval)
- [ ] Day 15: Address any issues from production test
- [ ] Day 15: Send Weekly Status Update

**Client touchpoint:** Feedback call (Day 11), async updates as needed, status email (Day 15)

---

### Week 4: QA, Documentation, Handoff

**Goals:** Bulletproof the system. Document everything. Hand it off clean.

- [ ] Day 16: Final bug fixes from Week 3 testing
- [ ] Day 17: Run full QA checklist (see qa-checklist.md)
- [ ] Day 18: Write documentation and create walkthrough recording
- [ ] Day 19: Client walkthrough call, collect final feedback, make adjustments
- [ ] Day 20: Final handoff
  - [ ] Send handoff document
  - [ ] Transfer all credentials and access
  - [ ] Send final invoice
  - [ ] Confirm support period
  - [ ] Send final Weekly Report

**Client touchpoint:** Walkthrough call (Day 19), handoff (Day 20)

## Risk Checkpoints

| Checkpoint | Date | Question to Answer |
|-----------|------|-------------------|
| Week 1 Review | Day 5 | Are requirements clear? Any missing access? |
| Week 2 Review | Day 10 | Is the core build on track? Any scope concerns? |
| Week 3 Review | Day 15 | Are we ready for QA? Any blockers? |
| Go/No-Go | Day 17 | Does it pass QA? Are we ready to hand off? |

If the answer to any checkpoint question is "no," escalate to Harper within 4 hours.

---

## Buffer

Built-in buffer: 2 days. Days 19-20 include buffer for final adjustments. If the project is at risk of slipping, communicate by Day 14 at the latest. Never surprise the client in Week 4.
