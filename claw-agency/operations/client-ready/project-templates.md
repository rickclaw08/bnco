# Project Management Templates by Tier

## Template Philosophy

Each tier has different expectations for speed, scope, and communication. These templates are starting points to be customized per client, not rigid requirements.

**Key Principles:**
- Under-promise, over-deliver on timeline
- Ship early, iterate fast
- Communicate blockers within 4 hours
- No surprises for clients

---

## Starter Tier ($500 one-time)

### Overview
**Scope:** Small, well-defined project with clear deliverable
**Timeline:** 1-2 weeks
**Ideal For:** MVPs, prototypes, single features, proof-of-concepts

### Project Structure

**Week 1:**
- Day 1: Kickoff + discovery (2 hours)
- Day 2-4: Development + daily updates
- Day 5: First review + feedback
- Day 6-7: Revisions

**Week 2 (if needed):**
- Day 8-10: Final development
- Day 11: Final QA + polish
- Day 12: Delivery + handoff
- Day 13-14: Support buffer

### Milestones & Deliverables

**Milestone 1: Setup & Foundation (Day 1-2)**
- Deliverable: Architecture doc OR basic framework
- Format: Document + GitHub repo
- Client approval: Quick async review

**Milestone 2: Core Development (Day 3-7)**
- Deliverable: Working prototype/feature
- Format: Demo video + staging link/repo
- Client approval: Test and provide feedback

**Milestone 3: Polish & Handoff (Day 8-12)**
- Deliverable: Final version + documentation
- Format: Production-ready code + README + video walkthrough
- Client approval: Sign-off form

### Check-In Cadence

**Async Updates:**
- Daily: End-of-day summary in Slack (2-3 sentences)
- Mid-week: Demo video (30-60 seconds, phone recording is fine)

**Sync Meetings:**
- Kickoff: 60 minutes
- Mid-point check: 30 minutes (optional, based on complexity)
- Final handoff: 30 minutes

**Response Time Commitments:**
- Client questions: <24 hours (business days)
- Blocker escalations: <4 hours
- Feedback incorporation: <48 hours

### Deliverable Package

**Final Delivery Includes:**
1. Source code (GitHub repo with clean history)
2. README with setup instructions
3. Demo video (3-5 minutes)
4. Architecture overview (1-page diagram)
5. Known limitations document
6. 7-day bug fix support

### Budget Allocation

- Discovery & planning: 10% ($50 worth of time)
- Development: 60% ($300 worth of time)
- QA & polish: 20% ($100 worth of time)
- Documentation & handoff: 10% ($50 worth of time)

**Time estimate:** ~15-20 hours total

### Risk Management

**Scope Creep Protection:**
- Define "done" criteria in kickoff
- Document any additional requests as change orders
- 1 round of revisions included, additional rounds billed separately

**Common Blockers:**
- Client access delays: Build with mock data, swap later
- Unclear requirements: Document assumptions, get async approval
- Technical complexity: Flag immediately, propose simplified approach

### Success Criteria

- Delivered on time: <15 days from kickoff
- Client satisfaction: >4/5
- Scope adherence: <10% deviation from original spec
- Bug reports post-delivery: <3 issues

## Growth Tier ($2,000/month)

### Overview
**Scope:** Ongoing development, features, improvements, or continuous support
**Timeline:** Monthly retainer with weekly deliverables
**Ideal For:** Sustained product development, feature pipelines, technical partner relationship

### Project Structure

**Monthly Cycle:**

**Week 1: Planning & Kickoff**
- Monday: Month planning call (60 min)
- Tuesday-Friday: Sprint 1 execution
- Friday: Week 1 demo

**Week 2-3: Core Development**
- Monday: Sprint planning (30 min)
- Tuesday-Thursday: Development
- Friday: Weekly demo + retrospective (30 min)

**Week 4: Polish & Planning**
- Monday-Wednesday: Final development
- Thursday: Monthly review + next month planning (60 min)
- Friday: Buffer for overflow or documentation

### Milestones & Deliverables

**Monthly Deliverable Target:**
2-4 significant features OR 1 major feature + ongoing maintenance

**Weekly Milestones:**

**Week 1: Foundation**
- Deliverable: Technical design for month's work + first feature
- Format: Design doc + working code
- Client approval: Async review + Thursday check-in

**Week 2: Feature Development**
- Deliverable: 1-2 features completed
- Format: Staging deployment + demo
- Client approval: Testing + feedback

**Week 3: Advanced Features**
- Deliverable: 1-2 features completed
- Format: Staging deployment + demo
- Client approval: Testing + feedback

**Week 4: Polish & Review**
- Deliverable: Production deployment + documentation
- Format: Release notes + video walkthrough
- Client approval: Production acceptance

### Check-In Cadence

**Async Updates:**
- Daily: Quick Slack update (1 sentence) during active dev
- Monday: Week plan (bullet points)
- Wednesday: Mid-week progress report (2-3 sentences + screenshot/video)
- Friday: Weekly summary + demo video (3-5 minutes)

**Sync Meetings:**
- Weekly: 30-minute check-in (Fridays preferred)
- Monthly: 60-minute planning session
- Ad-hoc: Available for calls with 24h notice

**Response Time Commitments:**
- Urgent questions: <4 hours (business hours)
- Regular questions: <12 hours
- Bug reports: <6 hours acknowledgment, fix timeline provided
- Feature requests: <24 hours scoping response

### Deliverable Package

**Weekly Delivery Includes:**
1. Working features on staging environment
2. Demo video
3. Code pushed to main branch
4. Basic testing completed

**Monthly Delivery Includes:**
1. Production deployment
2. Comprehensive documentation update
3. Release notes
4. Regression testing report
5. Next month roadmap

### Hour Allocation

**Monthly hours:** ~25-30 hours (~$65-80/hour effective rate)

**Typical breakdown:**
- Planning & meetings: 15% (~4 hours)
- Development: 60% (~15-18 hours)
- Code review & QA: 15% (~4 hours)
- Documentation & communication: 10% (~3 hours)

### Scope Management

**Included:**
- Up to 4 feature requests per month (complexity-adjusted)
- Bug fixes for work we've delivered
- Minor changes and tweaks
- Technical consultation
- Architecture guidance

**Not Included (requires additional payment):**
- Major scope changes (>10 hours of work)
- Integration with new third-party systems (unless planned)
- Infrastructure work (unless specified)
- Work on parts of codebase we didn't build (unless maintenance plan)

**Change Request Process:**
1. Client submits request via Slack or email
2. We scope and estimate within 24 hours
3. If <2 hours, include in current sprint
4. If >2 hours, discuss priority and timeline
5. If >10 hours, create separate project proposal

### Monthly Planning Process

**Last Week of Month:**
1. Review current month accomplishments
2. Identify next month priorities (client leads)
3. We provide technical guidance and estimates
4. Agree on 3-5 key objectives for next month
5. Document in shared roadmap

**Planning Meeting Output:**
- Next month's roadmap (prioritized list)
- Known risks or dependencies
- Success metrics for each objective
- Tentative weekly breakdown

### Retention & Off-boarding

**Month-to-Month Terms:**
- Either party can cancel with 15 days notice
- Final invoice prorated to last day worked
- Knowledge transfer included (1 handoff call)

**Pause Option:**
- Can pause for 1 month with 7 days notice
- Resume anytime with 7 days notice
- Good for seasonal businesses or budget cycles

### Success Criteria

- Monthly velocity: 3+ features delivered
- Client satisfaction: >4.2/5
- Bug rate: <5% of shipped features
- Missed deadlines: <10% of commitments
- Client retention: >6 months average

## Enterprise Tier ($5,000 - $15,000/month)

### Overview
**Scope:** Major product development, complex systems, or full technical partnership
**Timeline:** 3-6 month initial engagement, ongoing relationships
**Ideal For:** Serious product builds, technical co-founder replacement, high-stakes projects

### Project Structure

**Monthly Structure:**

**Week 1: Strategic Planning**
- Monday: Sprint planning + architecture review (90 min)
- Daily: Development with daily standups (15 min)
- Friday: Demo + strategic review (60 min)

**Week 2-3: Intensive Development**
- Monday: Sprint planning (45 min)
- Daily: Development + standups (15 min)
- Wednesday: Mid-sprint sync (30 min)
- Friday: Demo + retrospective (60 min)

**Week 4: Quality & Planning**
- Monday-Wednesday: QA, documentation, polish
- Thursday: Monthly business review (90 min)
- Friday: Code review + next month roadmap (60 min)

**Quarterly Structure:**

**Month 1: Foundation**
- Architecture and technical design
- Core infrastructure setup
- First major features

**Month 2: Feature Velocity**
- High-output development phase
- User testing and iteration
- Performance optimization

**Month 3: Refinement & Scale**
- Polish and edge cases
- Documentation and handoff prep (if needed)
- Next quarter planning

### Milestones & Deliverables

**Milestone cadence:** Every 2-3 weeks for major deliverables

**Example 3-Month Project (New Product Build):**

**Month 1:**

**Milestone 1: Technical Foundation (Week 1-2)**
- Deliverables:
  - System architecture document (detailed)
  - Database schema and API design
  - Development environment + CI/CD
  - Authentication and core infrastructure
  - Basic UI framework
- Format: GitHub repo + Notion technical spec + Figma (if design included)
- Review: 90-minute deep-dive with technical stakeholders

**Milestone 2: Core Features Alpha (Week 3-4)**
- Deliverables:
  - 2-3 primary user flows implemented
  - Admin panel foundations
  - Testing framework
  - Deployment to staging
- Format: Working staging environment + demo + test accounts
- Review: 60-minute demo + feedback session

**Month 2:**

**Milestone 3: Feature Completeness (Week 5-6)**
- Deliverables:
  - 80% of planned features implemented
  - Integration with key third-party services
  - Error handling and edge cases
  - Performance baseline established
- Format: Beta-ready staging environment
- Review: 90-minute demo + user testing observations

**Milestone 4: Beta Release (Week 7-8)**
- Deliverables:
  - 100% of MVP features
  - Security audit completed
  - Load testing results
  - User documentation (beta)
- Format: Production-ready beta environment
- Review: 60-minute stakeholder demo + go/no-go decision

**Month 3:**

**Milestone 5: Production v1.0 (Week 9-10)**
- Deliverables:
  - Production deployment
  - Monitoring and alerting
  - Incident response runbook
  - User-facing documentation
- Format: Live production system
- Review: 90-minute launch retrospective

**Milestone 6: Stabilization & Handoff (Week 11-12)**
- Deliverables:
  - Bug fixes from early users
  - Performance optimization
  - Knowledge transfer documentation
  - Video walkthrough series (architecture, deployment, maintenance)
- Format: Stable production system + complete docs
- Review: 120-minute handoff session + Q&A

### Check-In Cadence

**Async Updates:**
- Daily: End-of-day summary (3-4 bullet points) in dedicated Slack channel
- Monday: Week preview with priorities
- Wednesday: Mid-week progress video (2-3 minutes)
- Friday: Detailed weekly report (accomplishments, blockers, next steps)
- Monthly: Written executive summary with metrics

**Sync Meetings:**
- Daily standups: 15 minutes (optional, client choice)
- Weekly demos: 60 minutes (Fridays)
- Bi-weekly planning: 45 minutes
- Monthly strategic review: 90 minutes
- Quarterly roadmap planning: 2-3 hours

**On-Demand Access:**
- Slack: Monitored during business hours (9am-6pm client timezone)
- Emergency line: Phone number for production incidents
- Ad-hoc calls: Available same-day or next-day with Slack request

**Response Time Commitments:**
- Critical (production down): <1 hour, 24/7
- Urgent (blocker): <2 hours during business hours
- High priority: <4 hours
- Normal: <8 hours
- Low priority: <24 hours

### Deliverable Package

**Per-Milestone Delivery:**
1. Working software (staging or production)
2. Source code with clean commit history
3. Updated technical documentation
4. Demo video (10-15 minutes)
5. Test coverage report
6. Performance metrics
7. Known issues log
8. Next milestone preview

**Monthly Delivery:**
1. All milestone deliverables
2. Architecture diagrams (updated)
3. API documentation (if applicable)
4. Deployment runbook
5. Security review summary
6. Technical debt log
7. Executive summary presentation
8. Next month detailed plan

**Project Completion:**
1. Full source code with ownership transfer
2. Complete technical documentation
3. Video walkthrough series (10-15 videos, 5-10 min each)
4. Deployment automation
5. Monitoring setup
6. Incident response procedures
7. Maintenance recommendations
8. 30-day post-launch support

### Hour Allocation

**$5K/month:** ~60-70 hours (~$70-85/hour)
**$10K/month:** ~120-140 hours (~$70-85/hour)
**$15K/month:** ~180-200 hours (~$75-85/hour)

**Typical breakdown for $10K tier:**
- Meetings & planning: 10% (~12-14 hours)
- Core development: 65% (~78-91 hours)
- Code review & QA: 12% (~14-17 hours)
- Documentation & communication: 8% (~9-11 hours)
- Architecture & technical strategy: 5% (~6-7 hours)

### Team Composition

**$5K tier:**
- 1 senior dev agent (primary)
- Rick oversight (5-10% time)
- Harper coordination (5% time)
- Specialist agents as needed

**$10K tier:**
- 2 dev agents (senior + mid)
- Rick oversight (10-15% time)
- Harper coordination (10% time)
- Dedicated QA agent
- Specialist agents as needed

**$15K tier:**
- 3 dev agents (1 senior lead + 2 developers)
- Rick technical leadership (20% time)
- Harper project management (15% time)
- Dedicated QA agent
- DevOps/infrastructure agent
- Design agent (if needed)

### Scope Management

**Included:**
- All work within agreed project scope
- Reasonable scope refinements (<15% change)
- Bug fixes and quality issues
- Performance optimization
- Security best practices
- Technical consultation and strategy
- Architecture decisions
- Tool and technology selection

**Change Request Process:**
1. Client submits formal change request
2. We provide impact analysis within 48 hours (timeline, cost, alternatives)
3. Joint decision on priority: replace existing work, add to timeline, or defer
4. Updated SOW if budget impact >$2K
5. Revised timeline and milestones documented

**Flexibility vs. Control:**
- We maintain 10-15% buffer for unknowns and pivots
- Major direction changes require milestone reset
- We proactively suggest improvements but client has final say

### Quality Standards

**Code Quality:**
- Test coverage: >70% for critical paths
- Code review: All code reviewed before merge
- Security scan: Automated scanning + manual review
- Performance: Load tested to 2x expected traffic
- Documentation: Inline comments + external docs

**Process Quality:**
- Git workflow: Feature branches, PR reviews, semantic commits
- CI/CD: Automated testing and deployment
- Monitoring: Error tracking, performance monitoring, logging
- Backup: Automated daily backups with tested restore process

### Risk Management

**Weekly Risk Assessment:**
- Technical risks (complexity, unknowns, dependencies)
- Timeline risks (blockers, scope creep)
- Resource risks (availability, skill gaps)
- External risks (client dependencies, third-party APIs)

**Escalation Protocol:**
1. **Minor issue** (<1 day delay): Handle internally, note in daily update
2. **Moderate issue** (1-3 day delay): Slack notification + proposed solution
3. **Major issue** (>3 day delay or budget impact): Immediate call + written impact analysis
4. **Critical issue** (project at risk): Emergency meeting within 24 hours + recovery plan

### Success Metrics

**Project Health Indicators:**
- Milestone completion rate: >90% on-time
- Scope adherence: <20% variance from original plan
- Defect rate: <3% of features have major bugs post-delivery
- Client satisfaction: >4.5/5
- Technical debt: <15% of codebase
- Test coverage: >70%
- Documentation completeness: >90%

**Business Metrics:**
- Project completion rate: >95%
- Contract extension rate: >60%
- Referral rate: >30%
- Budget adherence: <10% variance

### Off-boarding & Transition

**Knowledge Transfer Process (2-4 weeks):**
1. Complete documentation audit
2. Record video walkthroughs
3. Live handoff sessions (3-5 sessions)
4. Q&A period (email/Slack for 2 weeks)
5. Emergency support option (1 month, limited hours)

**Transition Deliverables:**
- Complete codebase with documentation
- Access to all tools and environments
- Credential handoff (secure)
- Architecture decision log
- Future recommendations document
- Support contact for 30 days

## Cross-Tier Standards

### Communication Principles
- **Transparency:** Share progress and problems openly
- **Proactivity:** Don't wait for clients to ask
- **Clarity:** Avoid jargon, explain technical decisions
- **Responsiveness:** Meet or beat response time commitments

### Quality Gates
Every deliverable passes:
1. Code review (peer or lead)
2. Automated testing
3. Manual QA testing
4. Documentation check
5. Demo video recorded

### Documentation Standards
- README: Setup, architecture, deployment
- Code comments: Complex logic explained
- API docs: If applicable, OpenAPI spec
- Runbooks: How to operate and troubleshoot
- Decision log: Key architectural decisions and rationale

### Tool Stack
- **Code:** GitHub
- **Project management:** Notion or Linear
- **Communication:** Slack (primary), email (formal)
- **Demos:** Loom or Vidyard
- **Design:** Figma (if applicable)
- **Docs:** Notion or GitBook

## Template Customization Guide

**For each new project:**
1. Copy appropriate tier template
2. Adjust timeline based on complexity
3. Define specific milestones and deliverables
4. Set meeting cadence with client preference
5. Identify client-specific risks
6. Document in project folder
7. Share with client for approval
8. Review weekly and adjust as needed

**Red flags requiring template adjustment:**
- Client has never worked with agency before (add more check-ins)
- Highly regulated industry (add compliance reviews)
- Distributed team on client side (add async options)
- Unclear requirements (add discovery phase)
- Legacy system integration (add buffer time)
