5. Scanner Accuracy Disclaimers

Important: Scanner Limitations

ClawOps Shield uses pattern-based detection to identify potential security threats in text-based files. You should understand the following limitations before relying on scan results:

5.1 Detection Method

Shield uses rule-based pattern matching and heuristic analysis to detect potential prompt injection, data exfiltration, social engineering, and other threat patterns. It does not use comprehensive static analysis, dynamic analysis, or formal verification methods.

5.2 Informational Only

All scan results are provided for informational purposes only. Shield is designed to assist you in identifying potential risks, not to provide a definitive security assessment.

5.3 False Positives and False Negatives

Scan results may include:

  • False positives: Flagging content as potentially malicious when it is not.
  • False negatives: Failing to detect content that is actually malicious.

A clean scan does not guarantee that a file is free from malicious content or security vulnerabilities.

5.4 Not a Substitute for Professional Security Audits

Shield is not a substitute for professional security audits, penetration testing, or comprehensive security assessments. For critical or high-risk environments, you should engage qualified security professionals.

5.5 Client-Side Scanning

Scans run entirely in your browser. Your files are not uploaded to or stored on ClawOps servers. While this protects your data privacy, it also means scan capabilities are limited to client-side pattern matching.
