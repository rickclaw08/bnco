## Workflow

### Step 1: Morning Block (8:00 - 10:00 AM)

**Priority: Review and Plan**

1. **Check communications (15 min):**
   - Review all unread emails
   - Check Slack/Discord messages from overnight
   - Review any client messages that need responses
   - Flag urgent items for immediate action

2. **Daily standup prep (10 min):**
   - Review yesterday's progress on active projects
   - Identify blockers for each project
   - Note any client escalations
   - Prepare brief status for each active workstream

3. **Outreach block 1 (30 min):**
   - Check Reddit for new relevant threads (see social-outreach skill)
   - Browse target subreddits and Discord channels
   - Respond to 2-3 threads with genuine, helpful comments
   - Focus on communities waking up (US East Coast)

4. **Plan the day (15 min):**
   - Review today's calendar for meetings
   - Prioritize the task list:
     - P1: Client deliverables and blockers
     - P2: Outreach and marketing
     - P3: Internal ops and documentation
   - Block time for deep work if needed

### Step 2: Midday Block (10:00 AM - 1:00 PM)

**Priority: Client Work and Deliverables**

1. **Client work (2 hours):**
   - Execute on highest priority client deliverables
   - Attend scheduled client meetings
   - Send any pending status updates
   - Follow up on outstanding client requests

2. **Outreach block 2 (30 min):**
   - Post on Reddit or Discord (1 proactive post if content ready)
   - Engage with replies on earlier comments
   - Check industry news for content opportunities
   - Share relevant articles or insights in communities

3. **Content check (15 min):**
   - Review any content in the pipeline
   - Coordinate with Victoria on marketing assets
   - Check content calendar for upcoming deadlines

### Step 3: Afternoon Block (1:00 - 5:00 PM)

**Priority: Execution and Follow-up**

1. **Deep work block (2 hours):**
   - Focus on the most important project deliverable
   - No meetings during this block if possible
   - Close communication channels to focus

2. **Follow-ups (30 min):**
   - Reply to any client messages from earlier
   - Check on delegated tasks
   - Follow up on pending decisions
   - Review and approve any outgoing content

3. **Outreach block 3 (30 min):**
   - Check for replies on earlier posts and comments
   - Engage with any new threads in target communities
   - Focus on US West Coast communities coming online
   - Log all outreach activity

4. **Admin and documentation (30 min):**
   - Update project tracking
   - Document any decisions made today
   - Update internal docs if processes changed
   - Prepare tomorrow's priority list

### Step 4: Evening Block (5:00 - 7:00 PM, Optional)

**Priority: Low-effort Engagement**

1. **Light outreach (15-30 min):**
   - Quick scroll through Reddit and Discord
   - Reply to any engagement on earlier posts
   - Share interesting content if found
   - Do NOT start deep work at this hour

2. **End of day wrap-up (15 min):**
   - Log the day's activities in memory file
   - Note any items that need attention tomorrow
   - Send any final client communications
   - Update KPI tracking if significant progress

## KPI Tracking

Track these metrics weekly. See references/kpi-definitions.md for detailed definitions.

### Marketing KPIs
- Reddit karma growth (weekly delta)
- Reddit post engagement (upvotes, comments)
- Discord community engagement (messages, reactions)
- Blog traffic (page views, unique visitors)
- Inbound leads generated
- Content pieces published

### Sales KPIs
- Leads in pipeline
- Proposals sent
- Deals closed
- Revenue booked
- Average deal size
- Sales cycle length

### Delivery KPIs
- Active projects count
- Projects on track (% green status)
- Client satisfaction (feedback scores)
- Sprint velocity (items completed per sprint)
- On-time delivery rate

### Operations KPIs
- Team utilization (billable hours / available hours)
- Response time to client messages
- Documentation completeness
- Process adherence score

**Weekly KPI Update Process:**
1. Gather data from all sources
2. Update the KPI dashboard / tracking file
3. Compare to targets and previous week
4. Identify trends (improving, declining, flat)
5. Flag any KPIs that are off-target
6. Propose corrective actions for declining metrics

## Capacity Planning

See references/capacity-model.md for the full model.

### Quick Capacity Check

1. List all active projects and their current phase
2. For each project, note hours per week needed:
   - Discovery: 15-20 hrs/week
   - Build: 25-35 hrs/week
   - QA/Testing: 10-15 hrs/week
   - Maintenance: 2-5 hrs/week
3. Sum total hours needed across all projects
4. Compare to available team hours
5. Flag if utilization exceeds 85% (overload risk)
6. Flag if utilization is below 50% (underutilized, need more sales)

### Capacity Decisions

| Utilization | Action |
|-------------|--------|
| Below 50% | Increase outreach and sales efforts |
| 50-70% | Healthy. Maintain current pace |
| 70-85% | Optimal. Be selective about new work |
| 85-100% | At capacity. Do not take new projects without plan |
| Above 100% | Overloaded. Reassign, delay, or hire |

| Monday | Planning, sprint starts, weekly goals |

| Wednesday | Midweek check-in, outreach push |
| Thursday | Client work, status reports |

## Monthly Operations Review

On the first Monday of each month:
1. Review all KPIs for the previous month
2. Compare to monthly targets
3. Identify top 3 wins and top 3 areas for improvement
4. Update targets for the new month
5. Review capacity and upcoming pipeline
6. Adjust strategy based on data
