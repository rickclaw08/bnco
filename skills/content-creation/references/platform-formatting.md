# Platform Formatting Guide

## Version: 0.1.0

## Discord

### Do
- Use bullet lists for structured information
- Use **bold** for emphasis
- Use `code blocks` for technical content
- Use > blockquotes for highlighting key points
- Keep messages under 2,000 characters
- Use line breaks for readability
- Wrap multiple links in angle brackets to suppress embeds: `<https://example.com>`

### Do Not
- Do NOT use markdown tables (they do not render in Discord)
- Do NOT use headers (# syntax) in channel messages (they render as giant text and look bad)
- Do NOT send walls of text without line breaks
- Do NOT use more than 3 embeds per message

### Example (Good)
```
**Weekly Update**

Here is what we shipped this week:
- Client reporting agent (live)
- Onboarding automation (testing)
- Dashboard redesign (in progress)

Next week we are focusing on the dashboard rollout.
```

### Example (Bad)
```
| Feature | Status |
|---------|--------|
| Reporting | Live |
| Onboarding | Testing |
```

## WhatsApp

### Do
- Use **bold** for emphasis
- Use CAPS sparingly for section labels
- Use line breaks liberally
- Keep messages concise
- Use emoji for visual markers (sparingly)
- Use numbered lists for sequences

### Do Not
- Do NOT use headers (# ## ### syntax, they render as plain text with hash marks)
- Do NOT use markdown tables
- Do NOT use code blocks (render inconsistently)
- Do NOT send long messages (split into multiple if needed)

### Example (Good)
```
WEEKLY UPDATE

**Completed:**
1. Client reporting agent (live)
2. Onboarding automation (testing)

**Next week:**
Dashboard rollout starts Monday
```

## Reddit

### Do
- Use standard markdown (Reddit supports it well)
- Match the tone of the subreddit you are posting in
- Use TL;DR at the top for long posts
- Use proper formatting for readability
- Add relevant flair
- Keep titles descriptive (no clickbait)

### Do Not
- Do NOT use excessive formatting
- Do NOT use emoji in serious subreddits
- Do NOT post the same content in multiple subreddits simultaneously

## Twitter / X

### Do
- Stay under 280 characters
- Use line breaks for readability
- Use 1-3 relevant hashtags
- Thread long content (each tweet under 280 chars)
- Start with a hook

### Do Not
- Do NOT use markdown (it does not render)
- Do NOT use excessive hashtags (3 max)
- Do NOT start with "I" (less engaging)

## Email / Newsletter

### Do
- Use HTML formatting
- Include alt text on all images
- Provide plain text fallback
- Keep subject lines under 50 characters
- Use preview text effectively
- Test on mobile before sending

### Do Not
- Do NOT rely on images for critical information
- Do NOT use background images (poor email client support)
- Do NOT use custom fonts (fallback issues)

## Blog / Website

### Do
- Use full markdown
- Use proper heading hierarchy (H1, H2, H3)
- Include meta descriptions (150-160 characters)
- Add alt text to images
- Use internal links to other content
- Include a table of contents for posts over 1,500 words

### Do Not
- Do NOT skip heading levels (H1 to H3 without H2)
- Do NOT use H1 more than once per page
- Do NOT publish without a meta description
