# Contract Checklist

**Version:** 0.1.0
**Last Updated:** 2026-02-21

Use this checklist before sending any contract for review or signature. Every item should be checked and resolved.

---

## Pre-Send Checklist

### Identity and Parties

- [ ] All party names are correct legal entity names (not individuals unless appropriate)
- [ ] Addresses are current and correct
- [ ] Contact persons and titles are accurate
- [ ] Entity type is correct (LLC, Inc., Ltd., sole proprietor, etc.)
- [ ] All parties have legal capacity to enter the agreement

### Dates and Term

- [ ] Effective date is specified or tied to signature
- [ ] Term/duration is clearly defined
- [ ] Renewal terms are explicit (auto-renew? notice required?)
- [ ] Termination provisions are reasonable for both parties
- [ ] Notice periods are specified for termination
- [ ] Survival clauses identify which sections outlast termination

### Scope and Deliverables

- [ ] Scope of work is specific and measurable
- [ ] Deliverables are clearly defined with acceptance criteria
- [ ] Timeline and milestones are realistic
- [ ] Change order process is documented
- [ ] Out-of-scope items are explicitly excluded (if needed)

### Financial Terms

- [ ] Payment amount is clearly stated
- [ ] Payment schedule is defined (milestones, monthly, upon completion)
- [ ] Payment method is specified
- [ ] Currency is specified
- [ ] Late payment terms are included
- [ ] Expense reimbursement terms are clear
- [ ] Tax obligations are addressed (who pays sales tax, VAT, etc.)
- [ ] Invoicing process is defined

### Intellectual Property

- [ ] IP ownership is explicitly assigned
- [ ] Pre-existing IP is addressed
- [ ] License grants are specific (scope, exclusivity, territory, duration)
- [ ] Third-party IP is handled (licenses, attributions)
- [ ] Work-for-hire language is included (if applicable)
- [ ] Moral rights are addressed (if applicable in jurisdiction)

### Confidentiality

- [ ] Confidential information is defined
- [ ] Exclusions are reasonable
- [ ] Duration of confidentiality obligation is specified
- [ ] Return/destruction of materials is addressed
- [ ] Permitted disclosures are clear (employees, advisors, legal requirements)

### Liability and Risk

- [ ] Liability cap is included and reasonable
- [ ] Consequential damages are excluded (with appropriate exceptions)
- [ ] Indemnification obligations are mutual and balanced
- [ ] Force majeure clause is included
- [ ] Insurance requirements are specified (if applicable)
- [ ] Warranties are appropriate and not over-broad

### Legal and Compliance

- [ ] Governing law is specified
- [ ] Dispute resolution mechanism is defined (mediation, arbitration, litigation)
- [ ] Venue/jurisdiction for disputes is specified
- [ ] Assignment restrictions are included
- [ ] Severability clause is present
- [ ] Entire agreement clause is present
- [ ] Amendment process requires written agreement

### Formatting and Quality

- [ ] No placeholder text remaining (all brackets filled)
- [ ] Consistent formatting throughout
- [ ] Section numbering is correct and sequential
- [ ] Cross-references point to correct sections
- [ ] No typos or grammatical errors
- [ ] Defined terms are used consistently
- [ ] Signature blocks are complete and correct

## Risk Review Quick Check

### Red Flags (Must Fix)

| Check | Status |
|-------|--------|
| Unlimited liability exposure | [ ] OK |
| No termination clause | [ ] OK |
| Ambiguous IP ownership | [ ] OK |
| Missing governing law | [ ] OK |
| Unreasonable non-compete | [ ] OK |
| Auto-renewal without notice | [ ] OK |
| One-sided indemnification | [ ] OK |

### Yellow Flags (Review)

| Check | Status |
|-------|--------|
| Penalty clauses | [ ] OK |
| Exclusivity requirements | [ ] OK |
| Right to audit | [ ] OK |
| Most favored nation clause | [ ] OK |
| Non-solicitation scope | [ ] OK |

## Final Steps

1. [ ] Internal review complete (all checklist items addressed)
2. [ ] Risk flags resolved or documented with reasoning
3. [ ] Contract formatted for delivery
4. [ ] Summary of key terms prepared
5. [ ] Open questions documented for client
6. [ ] Sent to appropriate reviewer/approver before external delivery
