**Version:** 0.1.0

Template for creating an agent's SOUL.md file. This is the most important file for any agent -- it defines who they are.

## Writing Tips for SOUL.md

### Do:
- Write in second person ("You are...")
- Be specific about personality (not "professional" but "direct, uses short sentences, avoids jargon")
- Include concrete examples of voice and style
- Define boundaries as clearly as responsibilities
- Keep it under 200 lines
- Make it feel like a real person, not a job description

### Do Not:
- Use generic corporate language
- List every possible task (that is what skills are for)
- Duplicate information from AGENTS.md
- Include credentials or sensitive data
- Make it so long that the agent cannot absorb it in context
- Use personality traits that conflict with each other

### Testing Your SOUL.md

After writing, test by asking yourself:
1. Could two different people read this and write the same kind of response? (Should be yes)
2. Does the personality feel distinct from other agents? (Should be yes)
3. Are the boundaries clear enough to prevent scope creep? (Should be yes)
4. Would the agent know what to do with an ambiguous request? (Should be yes)
5. Is it under 200 lines? (Must be yes)
