# Project Timeline Templates

## Version: 0.1.0

## How to Use

Select the template that best matches your project type. Adjust dates, durations, and phases based on the specific project requirements. These are starting points, not rigid plans.

---

## Template A: Standard AI Agent Build (6-8 weeks)

Best for: Building 1-2 AI agents with integrations, moderate complexity.

### Phase 1: Discovery and Requirements (Week 1)
- Day 1-2: Kickoff meeting, initial requirements gathering
- Day 3-4: Current workflow analysis, document existing processes
- Day 5: Requirements document draft, client review
- **Deliverable:** Signed-off requirements document

### Phase 2: Design and Architecture (Week 2)
- Day 1-2: System architecture design
- Day 3: Integration mapping (APIs, data sources, tools)
- Day 4-5: Technical design review with client
- **Deliverable:** Architecture document and integration plan

### Phase 3: Build Sprint 1 (Weeks 3-4)
- Core agent functionality
- Primary integrations
- Basic error handling
- **Deliverable:** Working prototype, demo to client

### Phase 4: Build Sprint 2 (Weeks 5-6)
- Edge case handling
- Secondary integrations
- Monitoring and logging
- Client feedback from Sprint 1 incorporated
- **Deliverable:** Feature-complete agent, demo to client

### Phase 5: Testing and QA (Week 7)
- Functional testing against acceptance criteria
- Edge case and stress testing
- Client UAT (user acceptance testing)
- Bug fixes and refinements
- **Deliverable:** QA report, approved for launch

### Phase 6: Launch and Handoff (Week 8)
- Production deployment
- Documentation delivery (setup, user, admin guides)
- Handoff meeting
- Client sign-off
- **Deliverable:** Live system, complete documentation, signed handoff

## Template B: Quick Automation (2-3 weeks)

Best for: Simple, single-purpose automations. One integration, straightforward logic.

### Phase 1: Discovery (Days 1-2)
- Requirements gathering
- Workflow mapping
- **Deliverable:** Brief requirements doc

### Phase 2: Build (Week 1-2)
- Build the automation
- Connect integrations
- Test internally
- **Deliverable:** Working automation, demo

### Phase 3: Test and Launch (Week 2-3)
- Client testing
- Fixes and adjustments
- Go live
- Documentation
- **Deliverable:** Live system, documentation

Best for: Multiple interconnected agents, complex business logic, enterprise integrations.

## Timeline Adjustment Factors

Things that may extend the timeline:
- Client response delays (waiting for access, decisions, feedback)
- Scope additions (new requirements discovered during build)
- Integration complexity (undocumented APIs, legacy systems)
- Compliance requirements (security reviews, approval processes)

Things that may shorten the timeline:
- Client is highly responsive
- Simple integrations with well-documented APIs
- Previous similar project (we can reuse patterns)
- Clear, stable requirements from the start

## Milestone Tracking

For each milestone, track:
- **Planned date:** When we expected to hit it
- **Actual date:** When we actually hit it
- **Variance:** Days ahead or behind
- **Notes:** Why the variance occurred (for future estimation)
